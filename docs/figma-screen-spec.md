## Screen 0 — Issue 아카이브 (메인 진입점)

**목적**: 사이트의 첫 인상. "이 플랫폼은 계속 쌓인다"는 것을 전달. Issue 목록이 아카이브처럼 누적되는 구조.

**필수 요소**

| 영역 | 내용 | 디자인 방향 |
|------|------|------------|
| Navigation | 로고(좌) + Issue / Projects / People / About (우) | 얇은 선 또는 공백으로만 구분, 버튼 없음 |
| 플랫폼 한 줄 소개 | "전략·디자인·데이터의 교차점에서 프로젝트를 수행하고 아카이빙합니다" | 작은 sans-serif, 회색 |
| Issue 목록 | Issue 번호 / 주제 / 발행 연월 / 아티클 수 — 행(row) 형태로 나열 | 표처럼 수평 정렬, 클릭 시 해당 Issue 페이지로 이동 |
| 현재 Issue 강조 | Issue 00이 가장 위, 나머지는 추후 추가될 자리임을 암시 | Issue 00만 활성화, 나머지는 희미하게 또는 빈 행 |
| Footer | 팀명 / 연락처 / 발행 주기 안내 ("반기 발간") | 심플하게 |

**레이아웃 스케치**
```
PAIRED & PREPARED                    Issue  People  About

전략·디자인·데이터의 교차점에서 프로젝트를 수행하고 아카이빙합니다.

────────────────────────────────────────────────────────
No.    주제                              발행      아티클

00     차량 산업 패러다임의 변화          2026.06    6

## Screen 1 — Issue 00 페이지

**목적**: Issue 00 전체 구조를 한 눈에 파악. 4개 섹션으로 분기. 아티클 진입점.

**필수 요소**

| 영역 | 내용 | 디자인 방향 |
|------|------|------------|
| Issue 헤더 | "Issue 00" + "차량 산업 패러다임의 변화" (대형 타이포) + 발행일 + 참여 인원 | 전체 너비, 타이포그래피만으로 임팩트 |
| 에디터 인트로 | 서론 핵심 문장 발췌: "F1은 모빌리티 산업 전체의 축소판이다" | 1~2문장, 이탤릭 serif |
| 섹션 구분 목차 | Section 1~4를 명확히 구분해서 나열 | 섹션 헤더 + 그 아래 해당 아티클 행 나열 |
| 아티클 행 | 번호 / 제목 / 저자 / 예상 읽기 시간 | 줄 형태, 클릭 시 Article 상세로 이동 |
| 선배 인터뷰 블록 | 오프닝 인터뷰 / 클로징 인터뷰 — 별도 블록으로 시각적 구분 | 배경색 살짝 다르게 (예: `#EEEDE8`) |
| 부록 | 케이스 스터디 1, 2 링크 | 하단, 작게 |

**4개 섹션 구조**
```
Section 1.  Business
            자동차 산업 역사·공급망·기업혁신

Section 2.  Business × Design
            2-a. Feeling Luxury — 자동차 브랜드가 '좋은 차'를 정의하는 방식
            2-b. 전기차 과연 경제적인가 — 하이브리드 및 전기차 TCO 분석

Section 3.  Business × Engineering
            3-a. 피할 수 없다면 이용해라 — 친환경 정책으로 돈 버는 기업들
            3-b. AI 모빌리티 — AI 기반 배차·가격·수요 예측

Section 4.  최종 제언
            스스로 생각하고 달리는 차가 온다 — SDV·자율주행·플랫폼 전쟁
```

## Screen 2 — Article 상세 (아티클 읽기 화면)

**목적**: 개별 아티클을 에디토리얼하게 읽는 경험. 데이터·분석 콘텐츠가 많아 가독성이 핵심.

**필수 요소**

| 영역 | 내용 | 디자인 방향 |
|------|------|------------|
| 아티클 헤더 | 섹션 태그 + 제목 (대형 serif) + 저자 + 날짜 + 예상 읽기 시간 | 상단 넉넉한 여백 |
| 본문 | 1컬럼, 중앙 정렬, max-width 720px | 줄간격 1.8 이상, 문단 간격 넉넉히 |
| 풀쿼트 블록 | 핵심 문장 강조 | 큰 serif 이탤릭, 본문보다 폰트 크게, 좌우 여백 없애서 breakout |
| 데이터 시각화 블록 | 차트/표 삽입 자리 | 본문 너비보다 넓게 breakout (max-width 960px) |
| 각주 | 인용 출처 | 하단 작은 sans-serif |
| 다음 아티클 | "다음으로 읽기 →" | 하단, 제목 미리보기 포함 |

**아티클별 필수 시각화 슬롯**

| 아티클 | 필요한 시각화 |
|--------|-------------|
| Feeling Luxury | 그랜저 / 테슬라 / 제네시스 이미지 비교 3슬롯, 네이버 검색량 추이 차트 |
| 전기차 경제성 | TCO 비교표, 감가율 그래프, 중고차 데이터표, 설문 군집 차트 |
| 친환경 정책 | 이벤트 스터디 그래프, 주가 분석 시각화, 설문 결과 |
| AI 모빌리티 | PSM 곡선 2개 (일반택시 vs AI최적화택시), 시나리오 비교표 |
| SDV·자율주행 | R&D 비율 추이 그래프, 2030 산업 구조도 |

## Screen 3 — People 페이지

**목적**: 팀원 소개. 신뢰감 형성. 누가 이 리서치를 했는지 보여줌.

**필수 요소**

| 영역 | 내용 | 디자인 방향 |
|------|------|------------|
| 헤더 | "우리는 이렇게 일합니다" 또는 팀 소개 한 줄 | 간결하게 |
| 팀원 목록 | 이름 / 역할 / 담당 섹션 / 담당 아티클 | 그리드 아님, 목록(list) 형태 |
| Issue별 참여 구조 | Issue 00 참여 인원 표시 | 추후 Issue가 쌓이면 Issue별로 볼 수 있게 |

**팀원 목록 (Issue 00 기준)**
```

강다연  —  [역할]  —  서론, Section 2-a
신지우  —  [역할]  —  Section 4
양대현  —  [역할]  —  Section 2-b, Section 4
유지영  —  [역할]  —  Section 2-b, Section 3-b
이채경  —  [역할]  —  서론, Section 3-a
